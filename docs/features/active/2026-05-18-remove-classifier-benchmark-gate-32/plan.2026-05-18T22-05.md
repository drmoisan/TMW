Update `scripts/powershell/PoshQC/settings/pester.runsettings.psd1` to drop coverage references to deleted scripts. Retain `tests/TaskMaster.Benchmarks` untouched and prove it still builds.

Fail-closed evidence rule: every Phase 0 baseline task, every verification task, and every coverage-bearing toolchain task must produce an evidence artifact at the path stated in the task. Missing artifact => BLOCKED, not PASS.

Evidence root: `docs/features/active/2026-05-18-remove-classifier-benchmark-gate-32/evidence/` with canonical subfolders `baseline/`, `qa-gates/`, `regression-testing/`, `other/`. No other evidence root is permitted.

- Bundled-mirror parity is enforced by python + pester contract tests; mirror updates happen in the same plan (Phase 5), not deferred.
- `tests/TaskMaster.Benchmarks` must remain compilable. Phase 7 includes an explicit `dotnet build` task on that project.
- The seven-stage toolchain loop must complete in a single clean pass after all mutations (Phase 7). Any auto-fix or failure restarts the loop from step 1.
- Repo-wide grep sweeps in Phase 7 must return zero hits except in evidence/, the plan itself, and any potential entries inside the feature folder. Any other hit blocks completion.
