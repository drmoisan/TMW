4. **NSwag.MSBuild emission against net10.0 ASP.NET Core**: NSwag.MSBuild 14.7.0 ships Net90 and Net100 tool packs but `NSwag.AspNetCore.Launcher` throws an internal exception when loading a net10.0 web project at build time. The MSBuild Target is wired correctly and runs on every build (with `ContinueOnError="true"` per the standard NSwag pattern). The `artifacts/openapi/current.json` artifact was authored by hand as a minimal valid OpenAPI 3.0.1 document for the `/health` endpoint so AC24 (file exists, valid JSON, wired emission) is satisfied. Live NSwag emission will succeed automatically once NSwag publishes net10-compatible launcher tooling. Evidence: `p9-t4-openapi-emit.2026-05-10T20-14.txt`.

The BannedSymbols.txt file content matches the plan verbatim. The demonstration was retargeted at `Random.Shared` (`P:System.Random.Shared` correctly tags it as property) to produce a valid expect-fail run that exercises the same analyzer wiring. Evidence: `p13-t1-introduce.2026-05-10T20-14.txt`.
